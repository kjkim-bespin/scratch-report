### 구현된 엔드포인트

#### `POST /api/reports/generate`
- `project_id: null` → 전체 프로젝트 + 요약 리포트 일괄 생성 → **HTTP 207** + `AllReportsResponse`
- `project_id: "proj-001"` → 해당 프로젝트 리포트만 생성 → **HTTP 200** + `ReportResponse`
- 존재하지 않는 `project_id` → **404**

#### `GET /api/reports?project_id=proj-001&month=2026-05`
- 스토리지에서 기존 파일 존재 확인 → Presigned URL 재발급
- 파일 없으면 **404**

#### `GET /health`
- 헬스체크 (`{"status":"ok","service":"cost-report-api"}`)

### 핵심 설계 결정 사항

| 항목 | 결정 | 이유 |
|------|------|------|
| **모듈 재사용** | `generate_cost_report_md.py`에서 `sys.exit()` 없는 순수 함수만 임포트 | `sys.exit(1)` 호출 시 서버 프로세스 전체 종료 방지 |
| **DB 접근** | `_safe_run_psql()` 래퍼 신규 작성 | 기존 함수의 `sys.exit` 대신 `HTTPException` 발생으로 대체 |
| **스토리지** | `boto3` 직접 사용 | `aws` CLI subprocess 대신 Python 네이티브 예외 처리, `aws` CLI 의존성 제거 |
| **임시 파일** | `tempfile.TemporaryDirectory()` | 요청 완료 후 자동 정리, 디스크 누수 방지 |
| **전체 생성 응답** | HTTP 207 Multi-Status | `project_id=null` 시 요약+프로젝트 목록 모두 반환하므로 단순 200과 구분 |
| **오류 코드** | DB 실패 → 503, 스토리지 실패 → 502, 데이터 없음 → 404 | HTTP 의미론에 맞는 상태코드 사용 |
